\doxysubsection{Detailed Description}
The machine topology class represents the hierarchical topology of a machine, including N\+U\+MA nodes, cores and P\+CI Devices. 

Various infomation of the machine are gathered with the help of the Hardware Locality library (hwloc).

This class also provides functionalities to bind objects in the topology to the execution objects. Binding can enhance performance by allowing data to be closer to the executing object.

See the hwloc documentation (\href{https://www.open-mpi.org/projects/hwloc/doc/}{\texttt{ https\+://www.\+open-\/mpi.\+org/projects/hwloc/doc/}}) for more detailed information on topology detection and binding interfaces.

\begin{DoxyNote}{Note}
A global object of \mbox{\hyperlink{classgko_1_1MachineTopology}{Machine\+Topology}} type is created in a thread safe manner and only destroyed at the end of the program. This means that any subsequent queries will be from the same global object and hence use an extra atomic read. 
\end{DoxyNote}
